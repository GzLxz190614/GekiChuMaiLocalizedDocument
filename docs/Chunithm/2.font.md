研究了下解压后数据流，`DDS`是塞在`svo`里的，`svo`塞在解压后数据里，所以要完全实现打包还要研究`svo`（SEGA怎么这么喜欢一个文件是几个文件的合体）

AI的分析文档：[RFZ 字体解包分析](.\Extra\4.rfz\1.rfz_unpack_spec)

[rfz_unpack.py](.\files\rfz\rfz_unpack.py)

想要自制字体，使用[Bitmap Font Generator](https://angelcode.com/products/bmfont)，导出时设置长宽均为`2048`，`Bit depth`选`32`，`Preset`选`White text with alpha`，`Font descriptor`用`Text`或`XML`都行，反正需要指定。`Texture`选`dds`，`Compression`选`None`。

做好的`DDS`再用[DirectXTex](https://github.com/microsoft/DirectXTex)的`texconv`转为`B4G4R4A4_UNORM`，指令如下。

```bash
texconv.exe SEGAHUMMING_32pt_3.dds -ft dds -f B4G4R4A4_UNORM -o output -m 1
```

完成后，把`fnt`与`DDS`放在一起，用[bmfont_to_rfz.py](.\files\rfz\bmfont_to_rfz.py)即可。

!!! info "使用方法"
```bash
python rfz_unpack.py <input.rfz> [out_dir]

python bmfont_to_rfz.py SEGAHUMMING_32pt_text.fnt --format text -o RFO_SEGAKAKUGOTHIC_DB_32pt.rfz --name "TT-UD新ゴ 簡体字 Gb4 M" --comment "TT-UD新ゴ 簡体字 Gb4 M 32pt"
```
