## 自制

想要自制字体，使用[Bitmap Font Generator](https://angelcode.com/products/bmfont)，导出时设置长宽均为`2048`，`Bit depth`选`32`，`Preset`选`White text with alpha`，`Font descriptor`用`Text`或`XML`都行，反正需要指定。`Texture`选`dds`，`Compression`选`None`。

!!! info

    生成的`fnt`里，`base`（调整字体整体上下位置）和`lineHeight`（行距调整）的值可能需要修改，否则会出现字体偏下，显示不全的问题。显示不全也可能是字体大小？不太清楚

做好的`DDS`再用[DirectXTex](https://github.com/microsoft/DirectXTex)的`texconv`转为`B4G4R4A4_UNORM`，指令如下。

（不转也行，不过解包器好像强制`B4G4R4A4_UNORM`，不转的话拿来解包可能会解不了`DDS`）

```bash
texconv.exe SEGAHUMMING_32pt_3.dds -ft dds -f B4G4R4A4_UNORM -o output -m 1
```

!!! info

    或者用我fork的BMFont来做：[GzLxz190614/Bitmap-Font-Generator](https://github.com/GzLxz190614/Bitmap-Font-Generator)，添加了16 Bit depth导出（不用`texconv`转格式了）和一键选该字体所有可用字体
    
    我自己合并的中文+日文的字表：[rfz_chars.txt](.\files\rfz\rfz_chars.txt)

完成后，把`fnt`与`DDS`放在一起，用[bmfont_to_rfz.py](.\files\rfz\bmfont_to_rfz.py)即可。注意，不覆盖point会导致有的位置不显示字体，可以查看`Extra`内的文档来确认每个`pt`对应的`point`的值

```bash
python bmfont_to_rfz.py <fnt> --format {text|xml} [--dds-dir DIR]
    [--name S] [--comment S] [--point N] [--glyph-margin N]
    [--font-base S] [--avts-version N] [--dump-dir DIR] [-o out.rfz]

示例：
python bmfont_to_rfz.py SEGAHUMMING_32pt_text.fnt --format text -o RFO_SEGAKAKUGOTHIC_DB_32pt.rfz --point 24
```

!!! warning

    因为代码限制，字体最多22766个字，超了游戏直接崩溃，详细请看`Extra`里的文档
    
    推荐根据`Extra`内文档，每个pt对应的`point`覆盖，例如`32pt`的为`--point 24`，这样就不会有的地方不显示字了
    
    要是字的边缘有黑线，增大`padding`（好像`space也行`）就行了
